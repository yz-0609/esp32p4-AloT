# ESP32P4-AIoT

基于 ESP32-P4 与云端智能能力规划的多模态 AIoT 智能环境控制面板工程。

本项目面向 ESP32-P4-WIFI6-Touch-LCD-4B / ESP32-P4 86 Panel 类硬件，目标是把 720x720 触摸屏、双路继电器、Wi-Fi/时间同步、环境感知界面、场景模式和后续语音/大模型能力整合成一套可演示的桌面环境控制面板。

当前 README 根据仓库内以下文档和现有源码整理：

- `../ESP32-P4多模态AIoT智能环境控制面板项目方案.md`
- `IMPLEMENTATION_PLAN_UI_WIFI.md`
- `main/`、`components/`、`ui_mockups_lvgl/` 当前工程内容

## 项目定位

项目最终目标是形成一条完整闭环：

```text
GUI / 语音输入
    -> 环境感知与状态读取
    -> 本地规则或云端大模型决策
    -> ESP32-P4 本地安全校验
    -> 双路继电器和本机功能执行
    -> GUI 显示与语音反馈
```

第一阶段实际控制对象聚焦在：

- 继电器 1：照明设备开关。
- 继电器 2：风扇、净化器或通风设备开关。
- 720x720 LVGL 触摸界面。
- Wi-Fi 远程联网与 SNTP 时间同步。
- 环境数据展示和本地舒适度规则。

## 已完成内容

### 1. ESP-IDF 工程与目标配置

- 工程目标为 `esp32p4`。
- 已配置 32 MB Flash、自定义分区表、8 MB factory app、7 MB SPIFFS storage 分区。
- 已启用 PSRAM、200 MHz PSRAM、PSRAM XIP、caps allocator。
- 已关闭 LVGL 官方 demo，当前入口改为项目自定义 UI。
- 已加入 `esp_wifi_remote`，用于 ESP32-P4 通过 ESP32-C6 remote Wi-Fi 联网。

### 2. LCD、触摸与 LVGL 基础界面

- `main.c` 使用 BSP 启动 720x720 显示。
- LVGL draw buffer 已调整为整屏双缓冲，并放入 PSRAM：

```c
.buffer_size = BSP_LCD_H_RES * BSP_LCD_V_RES,
.double_buffer = BSP_LCD_DRAW_BUFF_DOUBLE,
.flags = {
    .buff_dma = false,
    .buff_spiram = true,
    .sw_rotate = true,
}
```

- `main/CMakeLists.txt` 为 LVGL 设置了 PSRAM 动态内存分配器。
- UI 已经不是 `lv_demo_widgets()`，而是调用 `ui_start()` 启动项目界面。

### 3. 多页面 LVGL 触摸 UI

已完成 720x720 项目主界面，包含：

- 启动等待页：显示 Wi-Fi / SNTP 初始化状态。
- 首页：显示时间、日期、网络状态、温湿度、光照、空气质量、设备状态和快捷场景。
- 控制页：灯光继电器、通风继电器手动开关。
- 环境页：温度、湿度、光照、空气质量、舒适度评分、传感器链路状态。
- 场景页：学习、休息、离家、自动、手动、定时等场景入口。
- AI 页：展示用户语音、LLM 回复、执行策略、安全检查等占位流程。
- 设置页：网络状态、音量、屏幕亮度、RS485 参数和系统信息。

当前 UI 使用 `ui_state_t` 作为统一状态中心，页面切换、动态标签刷新、触摸事件和继电器控制都通过该状态层协调。

### 4. 双路继电器控制

已完成 `app_relay` 模块：

- 继电器 1：`GPIO32`，对应灯光。
- 继电器 2：`GPIO46`，对应风扇或通风设备。
- 支持初始化时关闭双路继电器，进入安全默认状态。
- 支持单路开关、双路同步设置和当前状态查询。
- UI 控制页、场景页和 AI 占位动作已经可以调用该模块同步硬件输出。

### 5. Wi-Fi 联网与 SNTP 时间同步

已完成 `app_wifi` 模块：

- Wi-Fi STA 模式启动。
- 基于 `WIFI_EVENT` 和 `IP_EVENT_STA_GOT_IP` 判断连接状态。
- 支持断线重试，最大重试次数为 10。
- 连接成功后通过 `ntp.aliyun.com` 进行 SNTP 时间同步。
- 设置时区为中国标准时间。
- 向 UI 提供联网状态、时间同步状态、格式化时间接口。

目前 Wi-Fi 凭据仍硬编码在 `app_wifi.c` 中，后续应迁移到 Kconfig、NVS 或配网页面。

### 6. 网络天气数据接入

已完成 `app_weather` 模块：

- 等待 Wi-Fi 连接后启动 HTTP 请求。
- 调用 Open-Meteo 接口获取上海当前天气。
- 解析温度、体感温度、湿度、天气代码、风速、风向、降水、云量和更新时间。
- 每 10 分钟更新一次，失败后 30 秒重试。
- 通过互斥锁保护最新天气数据。

当前 UI 暂时把天气数据映射为环境页展示数据：温度、湿度来自网络天气，光照和空气质量根据云量/风速做本地估算。这是 RS485 环境传感器接入前的演示替代方案。

### 7. 场景模式与本地规则

已完成基础场景状态机：

- 学习模式：打开灯光，根据温度和空气质量决定是否通风，设置 25 分钟专注时间。
- 休息模式：关闭灯光和通风，清空专注时间。
- 离家模式：关闭双路继电器。
- 自动模式：根据光照、温度和空气质量执行本地规则。
- 手动模式：由用户直接切换继电器。
- 定时模式：当前仅设置 30 分钟状态值，尚未实现真实倒计时任务。

同时已实现舒适度评分规则，综合温度、湿度、光照和空气质量计算 0-100 分。

### 8. UI 原型与视觉方向

`ui_mockups_lvgl/` 中保留了早期 HTML 原型：

- 启动页
- 首页
- 控制页
- 环境页
- 场景页
- AI 页
- 设置页

当前 C 版 LVGL UI 已经吸收这些页面结构，并在设备端实现了主要页面和触控流程。

## 当前启动流程

`app_main()` 当前执行顺序：

```text
NVS 初始化
    -> 双路继电器初始化并关闭
    -> 启动 LCD/LVGL，开启背光
    -> 启动项目 UI
    -> 创建 Wi-Fi 联网任务
    -> 创建天气更新任务
```

UI 在时间未同步时会停留在启动等待页，SNTP 同步完成后自动进入首页。

## 当前限制

- Wi-Fi SSID 和密码仍硬编码，公开仓库前需要移除或改为配置项。
- 天气数据来自 Open-Meteo，并不等价于室内 RS485 传感器数据。
- AI 页面目前是本地模拟动作，没有真实 ASR、LLM、TTS 或结构化 JSON 校验。
- 音量调节只更新 UI 状态，尚未绑定真实音频硬件。
- RS485 页面为参数展示，尚未实现 Modbus RTU 主站和传感器采集。
- 定时模式只更新状态值，尚未实现 RTC/软件定时任务队列。
- 本 README 仅整理当前源码状态，本次未执行编译、烧录或硬件验证。

## 后续开发建议

优先顺序建议如下：

1. 将 Wi-Fi 凭据迁移到 Kconfig 或 NVS，并增加配网/重配入口。
2. 接入 RS485 Modbus RTU 传感器，用真实室内数据替换天气演示数据。
3. 实现定时任务服务，完成延时关灯、延时通风和专注计时提醒。
4. 增加参数本地保存，包括音量、亮度、场景偏好和网络配置。
5. 接入音频播放，再逐步加入麦克风采集、ASR、TTS 和离线固定命令。
6. 实现大模型服务客户端、结构化 JSON 指令解析和本地安全校验。
7. 增加控制历史、环境趋势、异常报警和连续运行测试记录。
